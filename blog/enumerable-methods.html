Enumerables and Enumerable#map

11.20.2015

what are Enumerables?

To start, let's go over what Enumerables aren't:

  • not a type of object in Ruby (like strings, integers, or arrays)
  • not just one method you can call on an object

You might notice that word "enumerable", in English, parses as an adjective. "Enumerables" (which you read about in the Ruby docs) is actually a short way of talking about a noun: "Enumerable classes." To reformulate: Enumerables are classes in Ruby which are able to be enumerated. Enumerables encompass a set of classes in Ruby, which respond to set of predefined methods (most of which have to do with enumerateing... or counting).

So... what are Enumerables again?

  • an adjectival description of an object type which gathers smaller items (a 'collection class') in ruby
  • an adjective for collection classes which get access to a set of methods
  • those actions tend to count through the collection classes (e.g., enumerate them!)

Now we'll look at one example of a method which Enumerables get to use.

Enumerable#map and Enumerable#collect

Note: since Enumerables use limited list of methods, the header above an example of how you'd signify a general Enumerable method, rather than the specific destructive or non-destructive versions (.map! or .map).

The .map or .collect method iterates through each distinct member of a collection and alters each one. (The two methods are synonyms; you could use either if it makes your code more readable.) .map or .collect output is an array.

.map is excellent at taking an array and applying a new instruction to transform it. Of course, you have to provide the instruction in a code block (much as for .each). Here's an example:

arr = ["a", "b", "c", "d"]
=> ["a", "b", "c", "d"]
arr.map { |letter| letter.upcase }
=> ["A", "B", "C", "D"]

It's good to note that although .each acts the same way as .map, their outputs differ; .each more easily gathers information from an array, where as .map is better suited to altering the array.)
