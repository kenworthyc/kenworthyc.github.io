CSS Positioning Values

11.8.2015

why positioning

When you lay out a design for a webpage— whether or not you already have aspirations for a font choice and and click actions— you have to understand its underlying structure. HTML will provide the page's content, but HTML and CSS work together to give the page its look, and its legibility for people. Since humans understand visual information better when distinct topics are displayed separately, that's how most webpages are set up: organized into boxes. Positioning puts all those boxes in order so we can understand it.

the four values

Every CSS attribute is divided from its value with a colon and closed with a semicolon, like so: attribute: value;. Our relevant attribute, position, has 4 valid values:

  1. static
  2. relative
  3. absolute
  4. fixed

position: static;

Static positioning has also been called "normal flow" by Jon Duckett, for good reason: static is the default value assigned to the position attribute. You'll never need to type "position: static;" but it's good to remember that the linear layout of HTML is not changed by static positioning. Sidebar: This means that block-level elements (which get a line break) will have their own row, while inline elements will appear on the line of the previous element.

position: relative;

Relative positioning allows an element to be adjusted relative to its 'normal' position. Its location can be changed, but the start point wil always be where the element would fall in static positioning.

position: absolute;

Absolute positioning will position an element relative to its parent element's position. The child copies the parent: by default, the top-left of an absolutely positioned element will coincide with the top-left of its parent element. To make use of the element's absolute positioning, it needs position assignments like 'top: 25%;' or 'right: 50px;'. Since the child element is connected

position: fixed;

Fixed positioning is actually a type of absolute positioning which takes the viewport as its 'parent' element. Fixed-position elements do not move on the browser; other elements move under or around it.

structure and positioning

These four values for positioning your HTML elements create just about all webpages. Each value has its use. Fixed-position header bars (which stay visible at the top as you scroll down) are found from blogs to online stores to new sites; absolute, relative and static positioning spread visual interest all across the web. Tinker around and you'll find they can be infinitely, infuriatingly adjustable, and just as useful.
