Listing in Ruby: Arrays and Hashes

11.14.2015

Arrays and hashes are some of the most useful ways to store data in Ruby. Read on for a basic idea of the differences between the two.

arrays!

Arrays are lists! In arrays, you can list all types of objects in Ruby; strings, integers, booleans, and other arrays are all valid array elements. In addition, the order of the elements is enmeshed with the array itself. That there is an order is an integral part of array-ness.

Ruby keeps track of all the elements in an array. Each element in an array corresponds to a number. They are sequenced like this: 

animal_array = [ cheetah, squirrel, ostrich ]
  # index numbers: [ 0, 1, 2]

Ruby links the first element in an array to 0. To point to 'cheetah' in animal_array, you could type animal_array[0]. For squirrel, animal_array[1]... and on and on, as long as your array goes.

hashes!

Hashes are lists too, but can keep information in more complex forms than arrays can. Each item in a hash is paired, and together they're commonly called a 'key-value pair'. Key-value pairs are uniquely useful because they allow each element to categorize and correspond to their pair.

Even better, the programmer can define every single item in a hashâ€” meaning each and every key and value. Ruby's built-in system of indexing arrays is numerical, but its system of indexing hashes is created by the programmer. 

favorite_animal_foods = { cheetah: "gazelle",
                          squirrel: "acorn",
                          ostrich: "pizza"
                         }

In the hash above, we could call any one of the elements, or the element's pair, as long as we know the name. If you know the right commands, Ruby will let you look through any one of these animals, and tell you their preferred cuisine. That project is for another article, but it shows why hashes may be useful.
